## Анализ результатов тестирования функции умножения матриц (sgemm) в Vortex

График разультатов: ../data/result/sgemm_graph.pdf

По результатам тестов по измерению масштабируемости при увеличении числа ядер от 1 до 16, количество тактов уменьшается, а при cores=16 достигает наименьшего значения.

В диапазоне от 17 до 31 ядра в ходе экспериментов возникали ошибки, свидетельствовавшие о неудачной попытке разделить задачу между всеми ядрами (примеры ошибок находятся в директории: ../data/errors). Например, при запуске конфигурации с 18 ядрами, в итоговой статистике для двух из них были выведены отрицательные значения количества выполненных инструкций и затраченных тактов. Всего было найдено 29584 ошибки. При использовании 20 ядер у каждого было положительное значение, однако тест также завершился некорректно, при этом было обнаружено 259 ошибок. Также была замечена закономерность: меньше всего сбоев возникало при использовании количества ядер, кратного четырём. Последующие увеличения их количества (на 1, 2, 3) вызывали рост числа ошибок (сопоставление числа ядер и числа ошибок: ../data/errors/cores_and_errors.txt).

При дальнейшем увеличении числа ядер (начиная с 32), наблюдался рост количества тактов. Это свидетельствует о накладных расходах, затрачиваемых на корректную работу: обеспечение параллельного выполнения большего числа ядер, больше запросов к памяти, больше вычислений. Вследствие чего суммарное количество затраченных тактов превышает значение, полученное при 16-ти ядрах, а последующие увеличения будут сопровождаться ростом затрат на их корректную работу. Таким образом, для умножения матриц 512×512 при фиксированных параметрах (clusters=1, warps=8, threads=64, l2cache, l3cache) наименьшее количество тактов достигается при использовании 16 ядер.
